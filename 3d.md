## obj格式解析
 obj 纯文本文件 line by line， v行 vt行 vn行 f行 f行格式:vertex/texture/normal,可以只包含v v/vt 或者v//vn, mtllib用于关联.mtl文件
主要指令：
1. 顶点数据指令 v vt vn vp
2. 面指令f f v1 v2 v3 f v1/vt1 v2/vt2 v3/vt3 f v1/t1/n1 v2/t2/n2 v3/t3/n3
3. 材质相关指令 mtlib filename.mtl 指定材质库; usemtl material_name 某些面使用材质库中的material_name; usemap map_name使用纹理贴图 在mtl文件中通过newmtl创建命名的材质
4. 组和对象指令 g group_name， o object_name
5. 曲线、曲面指令 cstype deg curv surf

## 3d 建模软件 
3d 建模软件有很多
总体上可以分为Nurbs建模和mesh建模
1. Mesh是有polygon拼成的，近似于曲面，实际是很多曲面，随着细分次数的增加，曲面越接近真实曲面，但计算量也越大，放大后还是曲面，不具备高精度的尺寸控制，无法导出矢量模型 常见建模软件有C4D、Maya、3ds Max、Blender、ZBrush等
2. Nurbs是非均匀有理b样条，是基于数学公式的曲面，真正的曲面，可以导出矢量图，因其高精度，常用于工业设计领域，包括
   1. CAID 外观设计领域 如RHino Alias
   2. CAD类型 结构工程 结构建模，如UG CATIA SolidWorks AutoCAD

### 核心概念解释

*   **材质**： 定义了模型表面对光线如何反应的属性集合。它是一个**数据组合**，通常由多张**纹理**贴图和一个**着色器** 程序共同定义。
*   **纹理**： 本质上是一张**图片/贴图**，用于输入到材质的各个通道中，控制不同的视觉属性。
*   **PBR**： **基于物理的渲染**，是现代实时渲染和影视渲染的行业标准。它使用一套符合物理规律的材质模型和贴图，以确保在任何光照环境下都能获得逼真且一致的效果。

| :--- | :--- |
| **基础颜色/漫反射** | 定义材质的纯颜色，不包含任何光照或阴影信息。 |
| **法线** | 通过RGB颜色信息模拟表面微观的凹凸细节，在不增加模型多边形的情况下极大增强细节感。 |
| **粗糙度** | 控制材质表面的粗糙程度。值越高（越白），散射光越多，看起来越暗淡；值越低（越黑），反射越清晰。 |
| **金属度** | 定义表面是金属还是非金属。金属具有独特的反射特性。 |
| **环境光遮蔽** | 模拟在模型裂缝和褶皱处因光线无法充分反射而形成的柔和阴影，增加深度和真实感。 |
| **高度/置换** | 真正在渲染时改变模型的几何形状来表现凹凸，比法线贴图更耗资源但效果更真实。 |
| **不透明度** | 控制材质的透明程度，用于制作玻璃、树叶、布料等。 |

### 典型工作流程

1.  **素材获取**： 使用 **Quixel Megascans** 或 **Poliigon** 获取扫描资源，或用 **Substance Sampler** 从照片生成材质。
2.  **程序化创作**： 使用 **Substance 3D Designer** 创建复杂的、可重复使用的程序化材质。
3.  **纹理绘制**： 在 **Substance 3D Painter** 或 **ArmorPaint** 中，将材质赋予3D模型，并结合手绘、投影等方式进行细节调整和个性化创作。
4.  **贴图烘焙**： 使用 **xNormal** 或 **Painter内置的烘焙工具**，将高精度模型的细节烘焙成法线贴图等，应用于低精度模型。
5.  **最终应用**： 将制作好的材质和纹理集导入到 **Blender, Maya, Unreal Engine, Unity** 等3D软件或游戏引擎中，通过着色器进行最终渲染。

### 核心概念解释

- **绑定**：为静态3D模型创建一套可运动的数字骨骼（骨架）和控制器的过程。优秀的绑定应该：
  - **直观**：动画师可以轻松理解如何操控。
  - **灵活**：能够做出各种极限姿势而不穿帮。
  - **稳定**：在极端情况下也不会崩溃。
- **动画**：通过在不同时间点上设置关键帧，来定义骨骼和物体的位置、旋转和缩放，从而创造出运动幻觉的过程。
- **变形器**：用于在不改变模型拓扑结构的情况下，使其形状发生变化，常用于制作肌肉膨胀、布料飘动等效果。
- **非线性动画**：允许动画师将不同的动画片段（如走路、挥手、跳跃）像视频剪辑一样进行混合、分层和编辑，提供了极大的灵活性。

### 典型工作流程

1.  **建模**： 创建静态的3D模型。
2.  **绑定**：
    - **骨骼创建**：在模型内部放置骨骼。
    - **蒙皮**：将模型顶点与骨骼关联，定义骨骼如何影响模型变形。
    - **控制器创建**：为动画师创建干净、易用的控制曲线或物体。
    - **添加变形器**：用于增强肌肉、服装等次级运动效果。
3.  **动画**：
    - **关键帧动画**：动画师手动设置关键姿势。
    - **动作捕捉**：使用真实演员的表演数据来驱动角色。
    - **修复与细化**：在引擎或动画软件中调整和优化动画。
4.  **导出**： 将绑定好的角色和动画数据导出到游戏引擎或渲染农场进行最终合成与输出。

### 总结

- **行业标准**：对于大型影视项目，**Maya** 仍然是绑定和动画的核心平台，并常配合 **Advanced Skeleton** 等插件和 **Houdini** 的程序化绑定来解决复杂问题。
- **独立与新兴力量**：**Blender** 凭借其免费和全功能的特性，正在被越来越多的独立艺术家和小型工作室采用。
- **技术前沿**：**Houdini** 的程序化绑定（KineFX）代表了未来的技术方向，提供了无与伦比的灵活性和可控性。
- **实时趋势**：以 **Unreal Engine** 为代表的游戏引擎，正在通过其强大的实时动画工具，改变着过场动画和虚拟制片的制作流程。
